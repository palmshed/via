# Templates

## PR Title

Format: `type[scope] :: description`

Examples:
- `feat[crashlytics] :: integrate Firebase Crashlytics for crash reporting`
- `chore[firebase] :: temporarily remove Firebase Crashlytics due to Flutter bug`
- `fix[ui] :: resolve button alignment issue`

## Impact
- [ ] New feature
- [ ] Bug fix
- [ ] Breaking change
- [ ] Build / CI
- [ ] Refactor / cleanup
- [ ] Documentation
- [ ] Tests
- [ ] Performance
- [ ] Security

Impact validation rule:
- Before PR create/edit, verify each checked category has at least one matching Summary bullet.
- If no concrete change supports a category, keep it unchecked.
- Keep this validation step in place.
